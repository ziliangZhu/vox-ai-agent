# VoxAgent Public Roadmap

This roadmap describes public-facing directions for VoxAgent. It is intended for users, partners, developers, and community members who want to understand where the product is going.

The roadmap may change based on product validation, customer feedback, compliance requirements, and production readiness.

## Current Focus

- Clarify VoxAgent as a telephony runtime for AI agents.
- Improve public website pages for product evaluation and scenario understanding.
- Provide AI-readable documentation for search assistants, LLMs, crawlers, and RAG systems.
- Support evaluation flows from product overview to concepts, quickstart, API documentation, and trial application.
- Prepare clearer paths from sandbox validation to production access.

## Near-Term Directions

- Improve developer onboarding and first-call validation guides.
- Expand scenario pages for AI sales, AI customer service, medical follow-up, education, and professional services.
- Make integration boundaries clearer: what VoxAgent handles and what customers keep in their own agent stack.
- Improve FAQ coverage for trial access, sandbox resources, production readiness, and compliance preparation.
- Add more public examples for webhook events, outbound calls, and runtime lifecycle concepts.

## Longer-Term Directions

- Broader runtime support for complex phone agent scenarios.
- More structured public metadata for AI agents, search systems, and documentation crawlers.
- Better partner-facing material for ecosystem integrations.
- More operational guidance for production launch preparation.
- More transparent product updates through public docs and release notes.
